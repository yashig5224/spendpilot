# PROMPTS

## Overview

I used AI tools for:

* building user interface basics

* finding and fixing errors

* writing documentation

* making designs work well on different devices,

* simplifying code that was repeated.

The main tools I used were:

* Cursor

* Claude Sonnet

* ChatGPT

## Example Prompt 1

"Create a landing page for a software service using Next.js 15 TailwindCSS shadcn/ui and Framer Motion. The design should be basic work on all devices and look like Linear or Vercels design."

### Why I wrote it this way

I wanted the AI to focus on:

* making a user interface like a software service

* ensuring it works well on all devices

* keeping the code simple and clean

* using a design style.

## Example Prompt 2

"Make an audit tool, in TypeScript that calculates how much money AI software saves using set pricing rules not rules made by AI."

### Why I wrote it this way

I wanted the business logic to be reliable and easy to test not suggestions made by AI that could be unpredictable.

## Example Prompt 3

"Simplify this React component to reduce state and make it easier to read without changing how it works."

### Why I wrote it this way

I used AI mostly to clean up the code and remove parts that were not needed.

## What did not work well

When I gave AI general tasks it often:

* created complicated architecture,

* made unnecessary files

* added too many abstractions

* suggested unrealistic patterns.

I found that giving tasks got much better results.

## What I did not trust AI with

I did not rely on AI for:

* calculating prices

* audit logic,

* making big architecture decisions

* fixing errors in live issues

* setting up deployment.

I. Tested these areas myself because they were critical and had to be correct.
